# NodeBB 钉钉登录配置指南

## 当前状态

✅ NodeBB 已启动  
❌ 钉钉 SSO 未配置

## 快速配置步骤

### 第一步：获取钉钉应用凭证

1. 访问 [钉钉开放平台](https://open.dingtalk.com/)
2. 登录并创建应用（或使用现有应用）
3. 获取以下信息：
   - **Client ID**（ClientId 或 AppKey）
   - **Client Secret**（ClientSecret 或 AppSecret）
4. 配置回调地址：
   ```
   http://localhost:4567/auth/dingtalk/callback
   ```
   （如果是生产环境，替换为你的实际域名）

### 第二步：配置环境变量

**推荐方式：创建 .env 文件**

1. 复制 `.env.example` 为 `.env`：
   ```cmd
   copy .env.example .env
   ```

2. 编辑 `.env` 文件，填入你的钉钉凭证：
   ```
   DINGTALK_CLIENT_ID=dingxxxxxxxxxxxxxxx
   DINGTALK_CLIENT_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

3. 确保 `app.js` 加载了 dotenv（已配置则跳过）

**临时方式：命令行设置**

每次启动前在命令行执行：
```cmd
set DINGTALK_CLIENT_ID=你的ClientID
set DINGTALK_CLIENT_SECRET=你的ClientSecret
node app.js
```

### 第三步：激活插件

1. 访问管理后台：http://localhost:4567/admin
2. 进入 **Extend → Plugins**（扩展 → 插件）
3. 找到 `nodebb-plugin-sso-dingtalk`
4. 点击 **Activate**（激活）
5. 点击 **Rebuild & Restart**（重建并重启）

### 第四步：测试登录

1. 访问：http://localhost:4567/login
2. 应该能看到 "Login with DingTalk" 按钮
3. 点击测试钉钉登录

## 验证配置

### 检查环境变量是否生效

在命令行执行：
```cmd
echo %DINGTALK_CLIENT_ID%
echo %DINGTALK_CLIENT_SECRET%
```

应该显示你配置的值（不是 `%DINGTALK_CLIENT_ID%`）

### 检查插件日志

启动 NodeBB 后，查看日志中是否有：
- ✅ `[sso-dingtalk] Plugin initialized`
- ❌ `[sso-dingtalk] Missing required env vars`（说明环境变量未设置）

## 常见问题

### Q: 看不到钉钉登录按钮？

**A:** 检查以下几点：
1. 环境变量是否正确设置
2. 插件是否已激活
3. 查看 NodeBB 日志是否有错误

### Q: 点击登录按钮后报错？

**A:** 检查：
1. 钉钉开放平台的回调地址是否正确配置
2. `config.json` 中的 `url` 是否正确
3. Client ID 和 Secret 是否正确

### Q: 如何重启 NodeBB？

**A:** 
```cmd
# 停止（如果在运行）
Ctrl+C

# 重新启动
node app.js
```

或使用 NodeBB 命令：
```cmd
./nodebb restart
```

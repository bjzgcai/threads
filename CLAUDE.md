# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

### dev mode
```bash
npm run dev
```

### Running NodeBB
```bash
./nodebb start          # Start in production mode (via loader.js process manager)
./nodebb dev            # Start in development mode
node app.js             # Run directly without the process manager
```

### Build
```bash
./nodebb build          # Build all assets (JS, CSS, templates, languages)
./nodebb build js       # Build only JavaScript
./nodebb build css      # Build only CSS/SCSS
./nodebb build tpl      # Build only templates (Benchpress)
```

### Testing
```bash
npm test                        # Run full test suite (lint + mocha)
npx mocha test/topics.js        # Run a single test file
npx mocha test/topics.js --grep "pattern"  # Run specific tests matching a pattern
```

Tests require a running database (reads from `config.json`). The test mock (`test/mocks/databasemock.js`) uses the configured database but flushes it before each run.

### Linting
```bash
npm run lint            # ESLint with cache
```

### Setup / Upgrade
```bash
./nodebb setup          # Initial setup wizard
./nodebb upgrade        # Run upgrade scripts and rebuild
```

## Architecture

NodeBB is a Node.js forum application built on Express + Socket.IO, with pluggable database backends (MongoDB, Redis, PostgreSQL).

### Entry Points
- **`loader.js`** — Process manager, forks `app.js` and handles restarts/clustering
- **`app.js`** — Main entry point: loads config, starts web server or setup wizard
- **`nodebb`** — CLI script (`src/cli/`) for management commands (start, stop, build, upgrade, setup)
- **`src/webserver.js`** — Express app setup, middleware, route mounting

### Core `src/` Modules

The `src/` directory contains the backend, organized by domain:

- **`database/`** — Database abstraction layer. `index.js` loads the configured backend (`mongo/`, `redis/`, `postgres/`). All database calls go through this interface.
- **`plugins/`** — Plugin system: `hooks.js` implements the hook pipeline (`filter:*`, `action:*`, `static:*`, `response:*`). Plugins register hooks via `Plugins.hooks.register()`.
- **`routes/`** — Express route definitions. `index.js` mounts sub-routers. `routes/write/` contains the RESTful Write API endpoints. `routes/api.js` exposes read API endpoints.
- **`api/`** — Business logic layer for the Write API, consumed by both REST routes and Socket.IO handlers.
- **`socket.io/`** — WebSocket event handlers. Each namespace (topics, posts, user, etc.) maps socket events to `src/api/` calls.
- **`controllers/`** — Page controllers for server-rendered views. `controllers/write/` handles Write API requests.
- **`middleware/`** — Express middleware: authentication, CSRF, rate limiting, user session handling.
- **`activitypub/`** — ActivityPub/Fediverse federation implementation.
- **`meta/`** — Forum configuration, CSS/JS build pipeline, template compilation, settings.
- **`user/`, `topics/`, `posts/`, `categories/`, `groups/`, `messaging/`** — Domain models, each split into sub-files by concern.
- **`privileges/`** — Privilege checking for all content access.
- **`cache/`** — LRU/TTL cache utilities used throughout.

### Frontend (`public/src/`)
- **`public/src/client/`** — Page-specific client scripts
- **`public/src/modules/`** — Reusable client-side modules
- **`public/src/admin/`** — Admin Control Panel scripts
- **`public/src/app.js`** — Client-side app bootstrap

Templates use **Benchpress** (`benchpressjs`), compiled from `src/views/` (and theme directories) into `build/`.

### Plugin & Theme System
Plugins ship as npm packages (`nodebb-plugin-*`). They register hooks against the NodeBB hook system. Themes (`nodebb-theme-*`) provide Benchpress templates and SCSS that override or extend the base. Both are listed in `package.json` dependencies and activated in the Admin Control Panel.

### Configuration
- **`config.json`** — Main runtime config (database connection, URL, secret). Generated by `./nodebb setup`.
- **`nconf`** — Config library; values settable via `config.json`, environment variables (`__` separator), or CLI args.

### Code Style
- CommonJS modules (`'use strict'`; `require`/`module.exports`)
- Mostly AirBnB style; enforced by ESLint (`eslint.config.mjs`) using `eslint-config-nodebb`
- Async/await is preferred; legacy callback code is still present in some areas
- Server code is Node.js only; client code targets modern browsers

## DingTalk Integration

This deployment uses **DingTalk as the sole SSO provider**. The standard NodeBB local registration and local login flows have been replaced/disabled.

### Key Changes from Upstream NodeBB

- **Local login disabled** — `data.allowLocalLogin` is hardcoded to `false` in `src/controllers/index.js`. Users cannot log in with username/password.
- **Registration flow removed** — `Controllers.register` and `Controllers.registerInterstitial` have been removed from `src/controllers/index.js`. The `src/views/register.tpl` and `src/views/registerComplete.tpl` templates are deleted.
- **Registration middleware simplified** — `middleware.registrationComplete` in `src/middleware/user.js` is a no-op (calls `next()` immediately).
- **Login page** — Only shows DingTalk SSO button (via `alternate_logins` / `authentication` array). The "Alternative Logins" label is hidden (set to a space in `public/language/*/login.json`).
- **Client register script removed** — `public/src/client/register.js` is deleted.
- **DingTalk i18n key** — `"dingtalk": "Login with DingTalk"` added to `public/language/en-GB/login.json`.

### SSO Plugin Architecture

DingTalk login is implemented as a NodeBB SSO plugin (a `nodebb-plugin-*` package) that:
1. Registers a Passport strategy via the `filter:auth.init` hook
2. Provides `url` (login initiation) and `callbackURL` (OAuth callback) on the strategy object
3. The core `src/routes/authentication.js` (`Auth.reloadRoutes`) mounts these routes automatically

To add or update the DingTalk plugin, install it as an npm package and activate it in the Admin Control Panel under **Extend → Plugins**.
